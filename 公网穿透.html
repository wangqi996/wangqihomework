FRP 内网穿透终极部署协议

任务档案：建立一条从公网到内网Windows主机的稳定时空隧道，用于传输N8N服务数据流。规避一切环境陷阱，采用最稳健的 Docker 协议完成部署。

Phase 01: 公网中继节点部署 (Server-Side)

1.1 // 节点环境初始化: Docker引擎安装

为应对宿主机环境的不可预知性，我们选择部署标准化的容器引擎作为服务基石。

指令A：同步星际软件包数据库

sudo apt-get update

指令B：安装Docker.io引擎

sudo apt-get install docker.io -y

指令C：执行“Hello World”协议，验证引擎状态

sudo docker run hello-world

校验标准：控制台必须返回包含 Hello from Docker! 的确认信号。

1.2 // 防火墙矩阵配置: 端口矩阵授权

在云服务商（阿里云）的安全矩阵中，为数据流配置入站规则。每个端口都是一个独立的信道。

授权目标：创建三条独立的TCP入站规则，授权来源为 0.0.0.0/0 (任意位置)。

  • 信道 7000: 用于FRP核心通信链路。
  • 信道 7500: 用于FRP仪表盘监控界面。
  • 信道 8081: 分配给N8N服务的专用公网隧道出口。

1.3 // 启动FRPS服务容器: 一键部署协议

执行终极部署指令，该指令将自动从Docker Hub拉取稳定版镜像，并以后台守护模式启动服务容器。

sudo docker run --restart=always -d \
--name frps-server \
-p 7000:7000 \
-p 7500:7500 \
-p 8081:8081 \
snowdreamtech/frps:0.52.3

校验标准：执行 sudo docker ps，必须能观测到名为 frps-server 的容器，且其状态为 Up。

Phase 02: 内网客户端部署 (Client-Side)

2.1 // 客户端程序获取

从FRP官方发布矩阵下载适用于Windows (amd64) 的客户端程序包，并解压至本地目录。

2.2 // 终端安全协议配置

安全警报：Windows Defender可能会将FRP客户端误判为威胁。必须将FRP程序所在文件夹添加至“病毒和威胁防护”的排除项，以确保程序执行权限。

2.3 // 客户端配置文件校准

在FRP客户端目录中，创建名为 frpc.ini 的配置文件。这是客户端连接中继节点的唯一凭证。

[common]
server_addr = 你的-服务器-公网-IP
server_port = 7000

[n8n-service-tcp]
type = tcp
local_ip = 127.0.0.1
local_port = 5678
remote_port = 8081
校准注意：确保 server_addr 字段被替换为真实的公网IP地址。确认文件名后缀为 .ini，而不是被Windows隐藏的 .ini.txt。

Phase 03: 隧道激活与验证

3.1 // 激活客户端链路

打开Windows PowerShell，导航至FRP客户端目录，执行激活指令。

.\frpc.exe -c .\frpc.ini

校验标准：控制台必须返回 login to server success 和 start proxy success 的成功信号。

3.2 // 端到端数据流验证

在任意终端的浏览器中，访问以下时空坐标：

http://你的-服务器-公网-IP:8081

任务成功标志：目标N8N服务的界面被成功渲染。
